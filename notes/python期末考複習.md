## 1. 三大核心資料結構

### 列表 (List) `[]`：可變的有序序列

列表就像一個動態零件庫，你可以自由地新增、刪除或修改裡面的元素。

- **特性**：有序、允許重複、**可變（Mutable）**。
    

```python
# 宣告一個感測器數據列表
sensors = ["溫標A", "壓差B", "流量C"]

sensors.append("轉速D")  # 新增元素到尾端
sensors[1] = "新壓差B"   # 修改索引 1 的元素
print(sensors)          # 輸出: ['溫標A', '新壓差B', '流量C', '轉速D']
```

### 元組 (Tuple) `()`：不可變的有序序列

元組一旦建立，裡面的內容就絕對無法修改。它通常用於儲存結構固定、不希望被意外更動的系統常數或座標。

- **特性**：有序、允許重複、**不可變（Immutable）**。
    

```python
# 宣告一個不可變的設備極限參數 (上限值, 下限值)
limit_limits = (150.0, 20.0)

# 如果嘗試執行 limit_limits[0] = 160.0，系統會拋出 TypeError 錯誤（安全防線）
print(f"最大容許值：{limit_limits[0]}")
```

### 字典 (Dict) `{}`：鍵值對的對照表

字典透過「鍵（Key）」來快速查找對應的「值（Value）」，就像查閱規格表一樣，效率極高。

- **特性**：無序（Python 3.7+ 保持插入順序）、Key 必須唯一且不可變。
    

```python
# 宣告一個設備狀態字典
motor_status = {
    "id": "M-01",
    "rpm": 1500,
    "is_healthy": True
}

# 讀取與修改
print(motor_status["rpm"])       # 輸出: 1500
motor_status["is_healthy"] = False  # 更新狀態
motor_status["voltage"] = 220    # 新增鍵值對
```

## 2. 運算子與判斷子的結合

運算子負責產生布林值（`True` 或 `False`），並直接驅動 `if` 判斷式的走向。

### A. 比較運算子 (Comparison Operators)

用於數值或字串的定量對比：

- <code>==</code>（等於）、`!=`（不等於）、`>`（大於）、`<`（小於）、`>=`（大於等於）、`<=`（小於等於）。
    

### B. 邏輯運算子 (Logical Operators)

用於組合多個條件：

- **`and`**：所有條件皆為真，結果才為真（嚴格交集）。
    
- **`or`**：只要其中一個條件為真，結果即為真（寬鬆聯集）。
    
- **`not`**：反轉布林值（真變假，假變真）。
    

### C. 成員運算子 (Membership Operators)

專門用來檢查某個元素是否存在於 List、Tuple 或 Dict 中：

- **`in`** / **`not in`**
    

### 綜合實戰範例：自動化檢測邏輯

將資料結構、運算子與 `if` 完美結合：

```python
# 系統當前狀態
active_errors = ["E01", "E03"]  # 目前觸發的錯誤代碼列表
system_metrics = {"temp": 75, "current": 12.5}  # 數據字典

# 判斷邏輯一：使用成員運算子檢查是否有關鍵錯誤
if "E01" in active_errors:
    print("【緊急】偵測到核心錯誤 E01，系統即將安全停機！")

# 判斷邏輯二：使用比較與邏輯運算子檢查運行參數
if system_metrics["temp"] > 70 and system_metrics["current"] > 10.0:
    print("【警報】溫度與電流同時超標，觸發降頻保護。")
elif system_metrics["temp"] > 70 or system_metrics["current"] > 10.0:
    print("【提示】單項參數偏高，維持正常監控。")
```

剛剛我們學過了：
*   **函數 (Function)** = 一台專門處理某件事的機器（例如 `calculate_delta_v`）。
*   **模組 (Module)** = 一個裝著好幾台機器的房間。**（也就是一個 `.py` 檔案）**

那如果我們有幾十個房間呢？我們總不能把它們散落在一地吧？
這時候，我們就會拿一個**大箱子（資料夾）**把這些房間裝起來。

👉 **這個「裝著好幾個 `.py` 檔案的資料夾」，在 Python 裡就叫做「封包 (Package)」。**

### 📦 航太工廠的比喻

想像我們現在不是在做一台小無人機，而是在打造一台火星探測車。我們的專案資料夾長這樣：

```text
mars_rover/ (這是一個大封包 Package)
├── __init__.py      (這是一張名牌，告訴 Python「我是一個封包喔！」)
├── navigation.py    (導航模組)
├── life_support.py  (維生系統模組)
└── propulsion.py    (推進系統模組)
```

*(偷偷跟你說，在舊版的 Python 裡，資料夾裡面一定要放一個空的 `__init__.py` 檔案，Python 才會承認它是個封包。雖然現在的新版不加也可以，但很多工程師還是習慣放著當作標記！)*

### 📞 怎麼跨越封包打電話叫貨？

假設你現在在工廠的最外面（一個叫 `main.py` 的檔案），你需要用到「火星探測車 (mars_rover)」封包裡面的「導航模組 (navigation)」裡面的「計算路徑 (calculate_path)」這台機器。

寫法其實跟你剛剛寫的很像，只是要加上**資料夾的名字**，並用「小數點 `.`」把它們連起來：

```python
# 語法：from 資料夾名.檔案名 import 機器名
from mars_rover.navigation import calculate_path

# 叫進來之後，就可以直接用了！
path = calculate_path()
```

### 總結一下大小關係：
1. **封包 (Package)**：資料夾 (Folder)
2. **模組 (Module)**：檔案 (File)
3. **函數 (Function)**：檔案裡面的工具 (def)

So，當你以後看到人家寫 `from numpy.linalg import inv`，你就會看懂了！這代表他在 `numpy` 這個大資料夾裡，找到 `linalg.py` 這個檔案，然後拿出了 `inv` 這台機器。

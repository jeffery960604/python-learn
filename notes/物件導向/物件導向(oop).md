學習 Python 的**物件導向程式設計（Object-Oriented Programming, 簡稱 OOP）**，就像是學習一種「用人類認知世界的方式」來寫程式的方法。

在現實世界中，我們看到的是一個個的「物件」（例如：一輛車、一隻狗、一個人），這些物件都有自己的**特徵** and **行為**。OOP 就是把這種概念搬進程式碼裡。

為了讓你輕鬆理解，我們把 OOP 拆解成幾個最核心的概念，並搭配生活化的比喻與 Python 程式碼：

---

### 1. 類別 (Class) 與 物件 (Object)
這是 OOP 最基礎的概念。
*   **類別 (Class)**：就像是**「設計圖」或「模具」**。它定義了某種東西應該長什麼樣子、能做什麼事，但它本身還不是實體。
*   **物件 (Object)**：又稱為「實體 (Instance)」，是根據設計圖**實際做出來的東西**。

**🚗 比喻**：
「汽車設計圖」是**類別**。根據設計圖製造出來的「你的黑色 Toyota」和「我的白色 Honda」就是**物件**。

```python
# 建立一個汽車的「類別」(設計圖)
class Car:
    pass

# 根據設計圖，打造出兩個真實的「物件」
my_car = Car()
your_car = Car()
```

---

### 2. 屬性 (Attributes) 與 方法 (Methods)
設計圖裡面通常會包含兩樣東西：
*   **屬性 (Attributes)**：物件的**特徵/狀態**（名詞/形容詞）。例如汽車的顏色、品牌、目前的時速。
*   **方法 (Methods)**：物件的**行為/功能**（動詞）。例如汽車可以發動、加速、煞車。

**💡 兩個重要的 Python 關鍵字：**
1.  `__init__`：這是一個特殊方法（建構式），當你「創造」物件的那一瞬間，程式會自動執行它，用來設定物件的初始狀態。
2.  `self`：代表**「物件自己本身」**。在類別裡面定義方法時，第一個參數必須是 `self`，這樣程式才知道現在是哪一個物件在執行動作。

```python
class Car:
    # __init__ 是用來初始化物件屬性的方法
    def __init__(self, brand, color):
        self.brand = brand  # 屬性：品牌
        self.color = color  # 屬性：顏色
        self.speed = 0      # 屬性：速度，預設為0

    # 方法：加速
    def accelerate(self):
        self.speed += 10
        print(f"{self.color}的 {self.brand} 加速了！目前時速：{self.speed}")

# 創造物件
my_car = Car("Toyota", "黑色")
your_car = Car("Honda", "白色")

# 呼叫方法
my_car.accelerate()  # 輸出: 黑色的 Toyota 加速了！目前時速：10
```

---

### 3. 物件導向的四大核心特性
當你掌握了類別和物件，接下來就是 OOP 的靈魂所在——四大特性：

#### ① 封裝 (Encapsulation)
**概念**：把資料（屬性）和操作資料的工具（方法）包裝在一起，並且**隱藏內部的複雜細節，不讓外部隨便修改**。
**比喻**：開車時，你只需要知道踩油門會加速，不需要知道引擎內部怎麼運作；而且引擎被保護在引擎蓋下，你不能隨便碰到。
**Python 實作**：在屬性前面加上雙底線 `__`，就可以把它變成「私有屬性」。

```python
class BankAccount:
    def __init__(self, name, balance):
        self.name = name
        self.__balance = balance  # __balance 是私有屬性，外部無法直接看到或修改

    def get_balance(self):
        return f"{self.name} 的餘額是: {self.__balance}"

account = BankAccount("小明", 1000)
# print(account.__balance)  # 這行會報錯，因為餘額被「封裝」保護起來了
print(account.get_balance()) # 只能透過合法的方法來查看
```

#### ② 繼承 (Inheritance)
**概念**：子類別可以繼承父類別的屬性和方法，並且可以擴充自己的新功能。這大大增加了**程式碼的重複使用性**。
**比喻**：「電動車」也是一種「汽車」。所以電動車可以直接繼承汽車的「加速」、「煞車」功能，然後再自己加上「充電」的功能。

```python
# 父類別 (汽車)
class Car:
    def drive(self):
        print("車子往前開")

# 子類別 (電動車) 繼承了 Car
class ElectricCar(Car):
    def charge(self):
        print("電動車正在充電")

tesla = ElectricCar()
tesla.drive()  # 繼承自父類別的功能
tesla.charge() # 自己特有的功能
```

#### ③ 多型 (Polymorphism)
**概念**：不同的物件，可以使用**同一個方法名稱**，但表現出**不同的行為**。
**比喻**：如果我對一隻狗說「叫！」，牠會「汪汪」；如果我對一隻貓說「叫！」，牠會「喵喵」。指令一樣，但結果不同。

```python
class Dog:
    def speak(self):
        return "汪汪！"

class Cat:
    def speak(self):
        return "喵喵！"

# 建立一個通用的函數
def animal_sound(animal):
    print(animal.speak())

dog = Dog()
cat = Cat()

# 傳入不同的物件，呼叫相同的 speak()，會有不同的結果 (多型)
animal_sound(dog)  # 輸出: 汪汪！
animal_sound(cat)  # 輸出: 喵喵！
```

#### ④ 抽象 (Abstraction)
*(這個稍微進階一點，在 Python 中通常透過 `abc` 模組實現)*
**概念**：定義出一個「標準介面」，規定子類別「必須」實作某些方法，但父類別自己不去實作。
**比喻**：政府規定所有「交通工具」都必須具備「煞車系統」，但腳踏車、汽車、飛機的煞車原理完全不同。政府只管「你要有這功能」，不管「你怎麼實作」。

---

### 總結：為什麼要用物件導向？
如果你只寫幾十行的簡單程式，可能不需要 OOP。但當你要寫幾千行、甚至幾萬行的大型專案（例如遊戲、購物網站、資料分析系統）時，OOP 的好處就會顯現：
1.  **程式碼好整理**：一切都是物件，容易模組化。
2.  **不重複造輪子**：透過「繼承」可以重複使用寫過的程式。
3.  **好維護與團隊合作**：「封裝」讓程式碼不會互相干擾，A 工程師寫的類別可以安全地給 B 工程師使用。


# 組合

在物件導向程式設計（OOP）中，**「組合」（Composition）** 是一個非常核心且實用的概念。

在程式設計界有一句很有名的金科玉律：**「多用組合，少用繼承」 (Favor composition over inheritance)**。

要理解「組合」，我們通常會把它跟前面提到的「繼承」拿來做比較。簡單來說，它們代表了現實世界中兩種不同的關係：

### 1. 繼承 (Inheritance) = 「是一個 (Is-a)」的關係
就像前面提到的，**電動車「是一個」汽車**。
繼承代表著一種分類層級，子類別會完全繼承父類別的所有特性。

### 2. 組合 (Composition) = 「有一個 (Has-a)」的關係
**組合的意思是：一個複雜的物件，是由多個比較小的、簡單的物件「拼裝」而成的。**
例如：汽車「有一個」引擎、汽車「有四個」輪胎。
在程式碼中，**「組合」就是把其他類別的物件，當成自己類別的「屬性」來使用。**

---

### 🚗 生活化的比喻：組裝電腦
假設你要寫一個模擬「電腦」的程式：
*   如果用**繼承**的思維：你會先寫一個「電子產品」類別，然後讓「電腦」繼承它。
*   如果用**組合**的思維：你會認為「電腦」是由「CPU」、「記憶體」、「硬碟」這些零件拼裝起來的。電腦本身負責指揮這些零件來工作。

### 💻 Python 程式碼示範
我們把「汽車與引擎」的例子寫成程式碼，你就會秒懂：

```python
# 零件 1：引擎類別
class Engine:
    def start(self):
        print("引擎啟動：轟轟轟！")

# 零件 2：輪胎類別
class Tire:
    def roll(self):
        print("輪胎開始轉動...")

# 組合體：汽車類別
class Car:
    def __init__(self, brand):
        self.brand = brand
        # 【這就是組合！】
        # 汽車內部「擁有」了引擎和輪胎的實體物件
        self.engine = Engine()  
        self.tires = [Tire(), Tire(), Tire(), Tire()] # 四個輪胎

    def drive(self):
        print(f"準備駕駛 {self.brand}...")
        # 呼叫內部零件的功能來完成工作
        self.engine.start()       
        self.tires[0].roll()      
        print("車子往前開了！")

# 實際使用
my_car = Car("Porsche")
my_car.drive()

# 輸出結果：
# 準備駕駛 Porsche...
# 引擎啟動：轟轟轟！
# 輪胎開始轉動...
# 車子往前開了！
```

### 為什麼說「多用組合，少用繼承」？

你可能會想，既然「繼承」很方便，為什麼大家還推薦多用「組合」？這就像是**「血緣關係（繼承）」**與**「團隊合作（組合）」**的差別。

1. **繼承太死板（像血緣關係）**：
   如果你寫了一個 `鳥` 的類別，裡面有 `飛()` 的方法。然後你寫了 `企鵝` 繼承 `鳥`。結果慘了，企鵝不會飛！你必須要在企鵝類別裡把 `飛()` 的方法改掉或寫出錯誤警告。繼承階層一多，程式碼會變得牽一髮而動全身。

2. **組合很靈活（像樂高積木）**：
   如果用組合，我們可以寫一個 `飛行能力` 的零件，和一個 `游泳能力` 的零件。
   * 「老鷹」物件 = 組合了 `飛行能力`。
   * 「企鵝」物件 = 組合了 `游泳能力`。
   * 「鴨子」物件 = 同時組合了 `飛行能力` 與 `游泳能力`。
   
   如果哪天我想幫車子加上「導航系統」，我不需要去改寫整個車子的家族樹，我只要打造一個 `GPS` 物件，然後把它「組合」進我的車子裡就好了！

**總結來說：**
「組合」就是把物件當成積木，讓一個大物件把小物件包在自己的體內（當作屬性），透過呼叫這些小物件的功能，來完成複雜的任務。這種寫法可以讓程式碼更靈活、更好維護！

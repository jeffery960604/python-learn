在 Python 中，`def` 是 **define（定義）** 的縮寫，它的用途是讓你**「自訂函數（Function）」**。

如果說迴圈（`for` / `while`）是為了解決「重複執行相同的動作」，那麼自訂函數就是為了解決**「在不同地方，重複使用同一組程式碼」**的問題。

---

### 🧃 生活化的比喻：一台「自動果汁機」
想像你發明了一台果汁機：
1. **輸入（參數）**：你把「蘋果」丟進去。
2. **處理過程（函數內部的程式碼）**：果汁機開始切碎、攪拌。
3. **輸出（回傳值）**：最後機器倒出一杯「蘋果汁」給你。

在程式碼中，這台機器就是你用 `def` 打造出來的！

---

### 1. 最基礎的函數（無輸入、無輸出）
定義函數有幾個固定格式：`def` 關鍵字、函數名稱、小括號 `()`，以及最後的冒號 `:`。接下來縮排的區塊，就是這個函數要做的事情。

```python
# 1. 打造機器 (定義函數)
def say_hello():
    print("歡迎光臨！")
    print("今天天氣真好！")

# 2. 按下機器的開關 (呼叫函數)
say_hello()
say_hello()
```
**注意：** 打造好機器（定義）後，它不會自己動。你必須**「呼叫」**它（寫下 `say_hello()`），程式才會去執行裡面的動作。呼叫兩次，就會印出兩次。

---

### 2. 加上「參數 (Parameters)」：給函數的原料
一台只能說固定的話的機器不夠聰明。我們可以設計「投入孔」（參數），讓機器根據你丟進去的東西，做出不同的反應。

```python
# 小括號裡的 name 就是參數 (投入孔)
def greet(name):
    print(f"你好啊，{name}！")

# 呼叫函數時，必須傳入對應的資料
greet("小明")  # 輸出: 你好啊，小明！
greet("愛麗絲") # 輸出: 你好啊，愛麗絲！
```

你也可以設計多個投入孔，用逗號隔開：
```python
def make_juice(fruit, size):
    print(f"正在製作一杯 {size} 的 {fruit}汁...")

make_juice("蘋果", "大杯")  # 輸出: 正在製作一杯 大杯 的 蘋果汁...
```

---

### 3. 加上「回傳值 (Return)」：機器的產出物
這點是初學者最容易卡關的地方：**`print` 和 `return` 的差別！**
*   `print` 只是把結果**「顯示在螢幕上」**給你看，你看完就沒了。
*   `return` 則是把結果**「交還給程式」**，讓你可以把它存進變數裡，拿去做接下來的計算。

```python
# 這個函數負責計算兩個數字相加
def add_numbers(a, b):
    result = a + b
    return result  # 把計算結果交出來！(碰到 return，函數就會立刻結束)

# 把回傳的結果，存進變數 total 裡面
total = add_numbers(10, 5)

print(f"總和是：{total}")
print(f"總和乘以 2 是：{total * 2}") # 因為有 return，所以我們可以拿結果繼續做數學運算
```
*(如果上面的 `add_numbers` 只是 `print(result)` 而不是 `return`，那你試圖把它存進 `total` 時，會得到一個「沒有東西 (`None`)」，也就無法進行後續的乘以 2 了。)*

---

### 4. 進階實用技巧：預設參數 (Default Arguments)
有時候，你希望使用者如果忘記給原料，機器也能有一個「預設的處理方式」。我們可以在定義函數時，直接用等號 `=` 給參數一個預設值。

```python
# 預設 country 是 "台灣"
def user_info(name, country="台灣"):
    print(f"玩家名稱：{name}，來自：{country}")

user_info("小華")                # 沒有給國家，會使用預設值 -> 來自：台灣
user_info("John", "美國")        # 有給國家，就會覆蓋預設值 -> 來自：美國
```

---

### 💡 總結：為什麼我們要寫函數？

1.  **Don't Repeat Yourself (不要重複你自己)**：
    如果你發現程式裡有一段長度超過 5 行的程式碼，你在不同地方複製貼上了兩三次，那就代表你該把它包裝成 `def` 函數了。
2.  **程式碼看起來更乾淨 (好讀)**：
    把複雜的邏輯（例如：計算稅金、連接資料庫、爬取網頁）包裝成函數，主程式看起來就會像是一篇目錄一樣清晰。
3.  **好維護**：
    如果算稅金的公式改了，你不需要去程式的 10 個地方慢慢修改，你只要修改 `def calculate_tax():` 裡面的邏輯，所有呼叫這個函數的地方就會自動更新！
